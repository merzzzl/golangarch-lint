# GOLANGARCH

Architecture instruction for this repository, generated from `.golangarch.yml` by golangarch-lint.
Follow these rules when creating, moving or editing Go code. Run `golangarch-lint lint .` to verify.
Do not edit this file manually — regenerate it with `golangarch-lint docs`.

## Always enforced (no configuration)

- Every exported function or method must live in a file named exactly after it: `ReadModulePath` -> `read_module_path.go` (normalization: lowercase, underscores removed, `.go` stripped).
- Every unexported function or method must live in a file whose normalized name is a prefix of the function name: `configValidateRule` may live in `config.go`.
- The package name must be exactly equal to its directory name (`package main` is exempt).
- Every directory containing `.go` files must be covered by at least one rule below.
- `*_test.go` files are always ignored by all checks.

## Ignored paths (invisible to all checks)

- `**/*_test.go`
